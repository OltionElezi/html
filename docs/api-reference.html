📄 API Reference

REST API documentation for the InteractiveQ Spring Boot backend. Base URL varies by environment. All protected endpoints require a valid JWT or API key.

🔓 public No authentication required
🔒 JWT Requires Cognito Bearer token
🔒 admin Requires ROLE_admin
🔑 API key Accepts API key auth
GET POST PUT PATCH DELETE HTTP methods

🌎 Base URLs

EnvironmentBase URL
Developmenthttp://localhost:8080/

ⓘ Live Swagger UI

The API has full OpenAPI documentation available at {BASE_URL}/swagger.html. The raw OpenAPI JSON is at {BASE_URL}/api-docs. Swagger try-it-out is disabled in production.

📦 REST Controllers

The application has 4 explicit REST controllers plus a large number of Spring Data REST (HATEOAS) endpoints auto-generated from JPA repositories.

ControllerFileResponsibility
ContactController controller/ContactController.java Contact form submissions and management
PaymentSettingsController controller/PaymentSettingsController.java CRUD for payment gateway configurations (Saferpay, etc.)
SaferpayTransactionController controller/SaferpayTransactionController.java Saferpay payment transaction initiation & callbacks
FrontendController controller/FrontendController.java Frontend resource serving (deprecated / legacy)

🔗 HATEOAS / Spring Data REST Endpoints

Spring Data REST auto-exposes JPA repositories as HAL+JSON REST endpoints. These cover the bulk of the API. All require authentication (JWT or API key) unless listed in the public endpoints.

// Response format: application/hal+json (HATEOAS)
// Example response shape:
{
  "id": 1,
  "name": "Hotel Offer",
  "_links": {
    "self":    { "href": "/offers/1" },
    "offers": { "href": "/offers" }
  }
}

// Paginated collection response:
{
  "_embedded": { "offers": [...] },
  "_links": { "self": {...}, "next": {...}, "prev": {...} },
  "page": { "size": 20, "totalElements": 100, "totalPages": 5 }
}

Standard HATEOAS CRUD patterns per domain:

MethodPatternAction
GET/{resource}List all (paginated, sortable, filterable)
GET/{resource}/{id}Get one by ID
POST/{resource}Create new
PUT/{resource}/{id}Replace (full update)
PATCH/{resource}/{id}Partial update
DELETE/{resource}/{id}Delete
GET/{resource}/search/findBy*Custom search methods

📝 Request & Response Formats

Authentication Header

// JWT Authentication (primary method)
Authorization: Bearer eyJhbGciOiJSUzI1NiIsImtpZCI6Ii...

// API Key Authentication
// Header name is configurable — key value from application.yml
X-API-Key: Gustaffo_s8MPoyXljoWqo8wPQFNcB15vK...

Standard Request Headers

Content-Type: application/json
Accept: application/hal+json        # for HATEOAS endpoints
Accept: application/json            # for regular JSON endpoints

Pagination Query Parameters

# Spring Data REST supports these query params:
GET /offers?page=0&size=20&sort=createdAt,desc

# Response includes pagination metadata:
{
  "_embedded": { "offers": [...] },
  "page": {
    "size": 20,
    "totalElements": 150,
    "totalPages": 8,
    "number": 0
  }
}

Error Responses

// 401 Unauthorized — no/invalid token
HTTP/1.1 401 Unauthorized
WWW-Authenticate: Bearer realm="InteractiveQ"

// 403 Forbidden — authenticated but insufficient role
HTTP/1.1 403 Forbidden

// 404 Not Found — resource does not exist
HTTP/1.1 404 Not Found

// 400 Bad Request — validation error
HTTP/1.1 400 Bad Request
{
  "errors": [{ "field": "email", "message": "must not be blank" }]
}

ⓘ Note on HATEOAS discovery

Spring Data REST exposes a discovery endpoint at GET / (root) when authenticated. This returns all available resource links in HAL format, making the API self-documenting. The Angular frontend uses NgxHateoasClient to consume these HAL+JSON responses and automatically follow _links.
