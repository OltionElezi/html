🏠 InteractiveQ — Gustaffo Booking Platform

Enterprise hotel & hospitality booking management system with offers, experiences, payments, and loyalty features. Used by the Falkensteiner hotel group.

Java 17 Spring Boot 2.5 Angular 12 AWS Cognito PostgreSQL Production

📷 Backend Domain Modules

The backend is organized into domain modules inside com.gustaffo.reservations:

🎉

offers

Booking offer creation, management, customized & simple offer workflows

🌞

experiences

Experience product management, bookings, templates, eligibility policies

💳

payment

Payment processing, Saferpay integration, payment links & configuration

📃

confirmations

Booking confirmations, newsletter opt-in, confirmation email flows

📋

cms

Content management: QR codes, email templates, greetings, property data

🎟

asm

Activity & Ticketing System (ASM) — ticket widget, ticket management

✉

mail

Transactional email delivery via AWS SES, mustache template rendering

🔔

notifications

Push notifications and event-based notification system

🌠

loyalty

Guest loyalty program (module present, partially disabled in UI)

🏿

vouchers

Voucher management (module present, UI feature commented out)

👥

rbac

Role-Based Access Control — roles, features, entity permissions

⚖

connectors

External PMS/IBE connectors: Phobs, TravelClick, Apaleo, IATA

⚙

rules

Rules engine for business logic automation (UI commented out)

🔒

security

JWT auth, API key filter, Cognito integration, custom entry point

🔗 Project Connections

// Angular auth interceptor adds JWT to every request:
Authorization: Bearer <Cognito JWT access token>

// Spring Boot validates against Cognito issuer:
issuer-uri: https://cognito-idp.eu-central-1.amazonaws.com/eu-central-1_8Db4kY02L

// Cognito groups become Spring Security roles:
cognito:groups → ROLE_admin, ROLE_cms, ROLE_reservationmanager

// API response format: HAL+JSON (HATEOAS)
Content-Type: application/hal+json
