▶ Entry Point

The application starts from InteractiveQApp.java with scheduling enabled.

// interactiveq/app/src/main/java/com/gustaffo/reservations/InteractiveQApp.java

@SpringBootApplication
@EnableScheduling
public class InteractiveQApp {
    public static void main(String[] args) {
        SpringApplication.run(InteractiveQApp.class, args);
    }
}

ⓘ Running the backend

Navigate to interactiveq/app/ and run ./mvnw spring-boot:run or build with ./mvnw clean package then java -jar target/interactiveq-app-3.1.1.jar

🔒 Security Configuration

See Auth & Security for the full authentication flow. Summary of the security setup:

Three Authentication Methods

MethodHow it worksRoles granted
JWT (Cognito) Bearer token in Authorization header. Validated against Cognito issuer URI. Groups extracted from cognito:groups JWT claim. From Cognito user groups → ROLE_admin, ROLE_cms, ROLE_reservationmanager, etc.
API Key API key in request header, validated by APIKeyAuthFilter. Keys configured in application.yml. ROLE_admin, ROLE_web, ROLE_currency_config
Basic Auth (Ops) In-memory user (fmtg_manager). Password BCrypt-hashed. Used for /ops* endpoints. ROLE_admin

Security Rules (HttpSecurity)

// WebApplicationSecurityConfig.java — key rules:

// 1. CORS enabled, CSRF disabled (stateless JWT API)
http.cors().and().csrf().disable()
    .sessionManagement().sessionCreationPolicy(STATELESS)

// 2. Public endpoints — no auth required:
.antMatchers("/bookOffer*", "/offerRequest*", "/offerRequestWidget/**",
             "/emailTemplate/**", "/captcha/**", "/payments/**",
             "/notifications/**", "/confirmations/newsletter/**",
             "/p/cms/templates/**", "/landing/**",
             "/bookCustomizedOffer/**", "/sse/availableRoomTypes/**",
             "/cms/qr/**", "/asm/tickets/widget",
             "/swagger-ui/**", "/actuator/**").permitAll()

// 3. Admin-only endpoints:
.antMatchers("/ops*", "/ops/**").hasAuthority("ROLE_admin")

// 4. Everything else requires any authentication:
.anyRequest().authenticated()

JWT Authority Mapping

// Cognito JWT claim → Spring Security authorities
authoritiesClaimName = "cognito:groups"
authorityPrefix      = "ROLE_"

// Example: JWT contains cognito:groups: ["admin", "cms"]
// Results in Spring authorities: ROLE_admin, ROLE_cms
