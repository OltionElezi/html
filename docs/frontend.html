🔗 HTTP Interceptors

All HTTP requests pass through 4 interceptors in sequence before reaching the backend.

1st interceptor
AuthInterceptor
Fetches the current Cognito session from AWS Amplify (Auth.currentSession()) and adds the JWT access token as a Bearer Authorization header to every request.

Public URL bypass: Skips auth for routes containing: emailView, emailResponse, paymentResponse, newsletter, paynow, wb, oms.

Error handling: On 401 response, handles gracefully without loop.
2nd interceptor
HttpConfigInterceptor
Configures standard HTTP request headers and behavior for all requests. Sets content type and base HTTP options.
3rd interceptor
LoadingInterceptor
Shows a loading spinner/indicator when any HTTP request is in-flight. Hides the spinner when all pending requests complete.
4th interceptor
EnvironmentInterceptor
Dynamically replaces the API base URL in every request based on the currently selected environment. Reads the target environment from localStorage, enabling runtime API switching without a page reload.

🛣 Routing & Feature Guard

FeatureGuard

The FeatureGuard protects routes based on two factors:

  • Role check — user must have one of the required Cognito roles (from JWT groups)
  • Feature flag check — the user's entity must have the required feature enabled (e.g. simple_offer, user_management)
  • Redirects to /access-denied if either check fails

Runtime Environment Switcher

The header contains a dropdown allowing users to switch between environments at runtime:

EnvironmentAPI URL
Prod (Stable)https://booking.gustaffo.com/
Prod (Preview)https://interactiveq-prod-next.apps.gustaffo.com/

The selected environment is stored in localStorage. The EnvironmentInterceptor reads it and rewrites the apiUrl in every HTTP request — no page reload required.
